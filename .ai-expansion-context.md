# AI Expansion Context — Source of Truth für die 10X Routing Timeline

Dies ist das Blueprint-Dokument. Es definiert die Verzeichnisstruktur, in denen der gesamte Content injiziert werden soll.

## Project Overview
- **Project:** Schlüsseldienst Webseite
- **Ziele:** Enterprise-Level Skalierung von ~10 auf 50+ Routen. Zero-Breakage Integration.
- **Tech Stack:** Next.js (App Router), React, TypeScript (Strict), Tailwind CSS v4, Formspree, Calendly. Keine externen CMS erlaubt. Kein Sentry. Kein Dark Mode.

## Route Architecture (Planned Blueprint)
*(to be scaffolded under `/app`)*

### 1. Root Group
- `/page.tsx` (Homepage)
- `/layout.tsx` (Global)
- `/error.tsx`
- `/not-found.tsx`

### 2. Marketing Group `(marketing)`
Besitzt eigenes Layout für Marketing-Navigation und Footer.
- `/leistungen` (+ `/[slug]`)
- `/ueber-uns` (+ `/team`, `/philosophie`, `/geschichte`, `/karriere`)
- `/kontakt`
- `/termin-buchen`
- `/anfrage`
- `/danke`
- `/referenzen`
- `/case-studies` (+ `/[slug]`)
- `/partner`
- `/auszeichnungen`
- `/blog` (+ `/[slug]`, `/kategorie/[category]`)
- `/angebot/[slug]`
- `/aktion`
- `/ressourcen` (+ `/faq`, `/glossar`, `/checklisten`)

### 3. Legal Group `(legal)`
Besitzt kompaktes Layout ohne wuchtigen Header/Heroes.
- `/impressum`
- `/datenschutz`
- `/agb`
- `/widerruf`
- `/cookie-richtlinie`

## Data Models & Component Registry
- Design Tokens liegen in `app/globals.css` (Tailwind v4 `@theme`).
- Datenstrukturen werden in `/lib/data/*.ts` aufgebaut.
- Komponenten leben in strukturierten Sub-Directories innerhalb `/components/`.
